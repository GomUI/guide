🍥 메일 클라이언트는 사용자마다 다르다.

네이버, 다음, 네이트, Gmail, Outlook 2007, iPhone 메일 함…
요즘의 클라이언트들은 각종 다양한 환경에서 메일을 사용한다. 또한 버전 및 사용자의 환경설정에 따라 또 다르며,
특정 메일 클라이언트에서는 문제없이 보여지는 메일이라 하더라도 다른 메일 클라이언트를 사용자에게는 문제가 될 수 있다.
모든 메일 클라이언트에게 같은 UI를 보여주기는 사실상 힘든 일이지만 최대한 싱크를 맞추는것을 목표로 한다.


Ex) 각 메일 서비스 환경에 맞게 디자인 코딩 된 사례

메일 서비스 기업마다 제한하는 css코드가 있으므로 각각 환경에 테스트하여 수신 결과물에 문제가 없는지 확인 해야 함.

🍥 메일 html 코딩 시 문제가 있을 수 있는 CSS 속성

# 가급적 피해야 할 CSS 속성
background-color, background-image, background-position, background-repeat, border, border-collapse, border-spacing, bottom, caption-side, clip, cursor, clear, direction, display, empty-cells, float, font-family, font-variant, height, left, letter-spacing, line-height, list-style-image, list-style-position, list-style-type, margin, opacity, overflow, padding, position, right, table-layout, text-indent, text-transform, top, vertical-align, visibility, white-space, width, word-spacing, z-index…


사실 엄밀히 말하자면 모든 CSS에 다 문제가 발생할 가능성 있음.
이 메일은 HTML문서가 아니기 때문에 정상적으로 렌더링 될 거라는 보장이 없다.
Inline-css (HTML의 style attribute를 이용)의 경우는 상대적으로 잘 지원되지만, 이 경우에도 단축형은 지원하지 않는 케이스들이 있음.(#fff같은 경우)
결론 : inline-css를 제외하고는 CSS 사용에 문제 있음. 가능하면 CSS를 이용하여 디자인하지 말 것. Inline-css 에서도 모든 속성이 제대로 지원될 거라는 보장없음.

🍥 테이블 태그 사용

HTML 4.0 기준의 Table 관련 태그들은 제법 다양한 메일 클라이언트에서 지원되며,
css property를 사용함에 제한이 있으므로 table이나 td 태그의 html attribute (border, valign, align, cellspacing, cellpadding 등)을 이용하면 레이아웃을 잡을 수 있다.

권장사항은 아니지만, colspan이나 중첩 테이블 등의 경우 제대로 표시된다는 보장되는 것은 아니므로, 가능한 다단 편집 대신 1단으로 컨텐트를 구성하는 쪽이 바람직하다.

또한 table 관련 태그를 사용할 경우, width를 지정하겠다면 가능한 한 픽셀단위 고정 폭을 사용하지 말고, %단위의 가변 폭을 사용하는 쪽이 바람직하다. (사용자 환경을 알 수 없으므로 렌더링 될 넓이를 정확히 추측할 수 없기 때문.)



# 메일링 table 스타일 방식 
<td width="604" height="74" bgcolor="#f4f4f3" style="font-size:0; color:#ff6600;"><p style="font-size:18px;">...</p></td>
<td width="496"  style="text-align:left; padding-left:110px; line-height:14px; font-size:0px;">...</td>

폰트의 크기 및 컬러는 td에도 적용되지만, 서비스(네이버, 다음, 아웃룩 등) 환경에서 따라 reset스타일이 적용되어 디자인이 무너질 수 있다.
각각의 td안에 <p>태그나 <strong>태그를 사용하여 그에 inline_style로 안정된 디자인을 표현하는 것이 안전하다.
또한 line-height값을 지정해주지 않으면 서비스 환경마다 출력이 다르므로 필히 line-height 값을 지정해주어야 한다.



# 메일링 table 배경 이미지 사용 제한 
❌  <td ... style="background="test.jpg">...</td>
✔️  <td ...><img src="test.jpg" width="654" height="162" border="0" /></td>

가급적 background 코드는 사용 안하는 것이 안전하다.
만약 디자인 상 사용해야 하는 경우는 <img>코드로 삽입을 권장하며, background 코드는 아웃룩에서는 지원이 안되어 출력이 되지않는다.
따라서 백그라운드 이미지를 굳이 사용하는 경우에는 "없어도 메세지 전달에 지장이 없는" 이미지 여야 한다.
또한 모든 이미지 경로는 절대경로를 권장한다.
이미지가 표시되지 않는 경우, 해당 이미지가 전달하려던 메시지가 안보이게 되므로, 반드시 alt 속성을 이용해서 대체 텍스트를 제공해야 한다.
또한, width, height 속성을 정확히 사용해서 이미지가 표시되지 않더라도 해당 영역을 확보함으로써 레이아웃이 깨지지 않도록 처리해줘야 함.
결론 : 브라우저의 이미지 표시 옵션을 끄고서도 메일의 내용이 이해될 수 있도록 해야 한다.

🍥 각종 embed 요소

플래시 등을 로딩하기 위한 embed 태그나 javascript 등은 사용금지. (특히, 버튼이나 링크 등을 javascript로 작성하지 말 것!)
결론 : embed, javascript.. 이런 거 전부 쓰지 말 것.

🍥 결론

1. 고객에게 반드시 전달되어야 할 모든 메시지는 반드시 텍스트로 만들어져야 한다.
2. 텍스트 밑에 백그라운드로 이미지를 까는 것은 상관 없지만 디자인 상 크게 지장 없는 한에서 사용가능 (단 CSS로 제어는 안된다고 가정해야 함.)
3. 일부 컨텐트가 반드시 강조되어야 해서 이미지로 메시지를 작성할 경우, alt, width, height를 반드시 작성해야 이미지가 표시되지 않는 환경에서도 사용할 수 있다.
4. 모바일 환경에서는 특히 이미지를 표시 안하는 경우가 디폴트나 다름없으므로 반드시 주의해야 한다.
5. script를 절대 금지.

- HTML메일에서 태그는 닫는다. 
❌  <br>, <img src="..." alt="...">
✔️  <br/>, <img src="..." alt="..."/>

- Outlook PC버전 둥근 버튼 만들기

Outlook PC버전에서는 border-radius가 먹지 않아 css로 둥근 사각형을 만들 수 없다.
Outlook PC버전에서도 둥근 사각형을 만들고 싶다면 VML언어를 사용하여 아래와 같이 구현이 가능하다.
VML을 좀 더 자세하게 알고싶다면 여기를 참고. 

<td ... style="...">
  <!--위 코드는 아웃룩 pc 버전 대응 / 아래는 그 외 모든 환경 대응 -->
  <!--[if mso]>
  <v:roundrect xmlns:v="urn:schemas-microsoft-com:vml" xmlns:w="urn:schemas-microsoft-com:office:word" href="#"
style="v-text-anchor:middle;..." arcsize="13%" stroke="f" fillcolor="#ff7600">
      <w:anchorlock/>
      <center style="...">
        버튼 텍스트
      </center>
    </v:roundrect>
  <![endif]-->
  <!--[if !mso]> <!-->
    아웃룩이 아닌 모든 환경에 대응 코드
  <!-- <![endif]-->
</td>

* !--[if mso], !--[if !mso]: Outlook 환경인지 아닌지 검사하는 조건문 * v:roundrect: 둥근사각형 모양
* arcsize: 둥근 사각형의 정도(border-radius라고 생각하면 될 것 같다. %로 기재)
* stroke, fillcolor: border와 background. stroke를 사용하는 경우에는 t가 아니라 strokecolor="border 색" strokeweight="border 두께(pt)" 를 넣어야한다.
* <w:anchorlock/>: <a>와 같은 기능을 하지만 href는 상위 v:roundrect에 넣어줘야 한다.
